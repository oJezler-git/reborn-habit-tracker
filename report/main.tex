\section{Introduction}

\epigraph{\enquote{Make it so easy you can't say no.}}{\textit{James Clear, Atomic Habits} \cite{clear2018atomic}}

Modern digital systems increasingly compete for human attention through algorithmic optimisation, yet provide little computational support for helping users regulate behaviour and build sustainable habits. Many widely used platforms prioritise engagement over user agency, contributing to inconsistent routines and behavioural instability.

This project, \textbf{REBORN}, aims to build an intelligent habit-tracking system that automates scheduling, analyses behavioural risk, and dynamically adapts difficulty. When software handles the heavy planning, users can focus their energy on execution.

\section{Analysis}

\subsection{Problem Definition}

% Describing how manual habit tracking is exhausting and current technology is not intelligent enough to reduce this problem.
Many people experience \emph{self-regulation failure} - they want to improve, but the cognitive effort needed to plan and continuously maintain habits often leads to burnout \cite{baumeister1996self, baumeister1998ego}. Technology is not only the largest source of procrastination, but also doesn't provide intelligent ways to reduce this problem \cite{sirois2016procrastination}. Habit formation, therefore, is both a computational challenge and a psychological one. Tools that can automate planning, predict high-risk periods, and adapt interventions in real time are therefore necessary.


\subsection{Users}
% Describing the primary users of the system and their needs.
% This includes: 
%  A level students
%  Professionals
%  Fitness enthusiasts
%  General users who want to improve their habits
The primary users of REBORN are students, especially those studying multiple A-Levels and struggling with procrastination and the mental load of planning habits. For this group, automation and predictive intervention allow them to focus on doing the habits instead of organisation.  

\vspace{0.5em}
The system is also intended for a larger audience, such as:
\begin{itemize}
    \item Professionals who need structured routines and work-life balance.
    \item Individuals maintaining regular fitness or health-related habits.
    \item Users interested in general self-improvement and long-term behaviour change.
\end{itemize}

All of these user groups experience forms of self-regulation failure, where manual habit tracking can lead to cognitive overload and burnout.

\subsection{Secondary Research}

% The current state of habit tracking apps, and why we need REBORN.
Current habit-tracking and productivity applications differ widely in how much computation they perform. Some are manual-input database systems, others are game-like behavioural frameworks. An analysis of these tools shows that most apps use psychological principles (loss aversion/variable reward schedules) instead of algorithmic reasoning.

To justify the development of REBORN, I will contrast existing tools with a theoretical \enquote{gold standard} of computational habit formation, and then find specific technical gaps in scheduling, prediction, and how systems adapt to user context over time.

\subsubsection{PCS Framework}
% Some theory on PCS (the strongest framework for habit formation).
The strongest computational framework for habit formation is the \textbf{Predicting Context Sensitivity (PCS)} method \cite{buyalskaya2023what}. Unlike the \emph{21-day myth}, this machine-learning methodology models habit formation as a continuous measure of context-sensitive predictability.

% Explain what LASSO is here (machine learning tool used in habit research)
Using \textbf{LASSO} (Least Absolute Shrinkage and Selection Operator) regression---a regularised machine-learning method that identifies the most predictive contextual variables while reducing overfitting---on datasets with 40 million+ observations, Buyalskaya et al.\cite{buyalskaya2023what} showed that habit formation timelines are domain-specific. The estimated times range from a few weeks for simple hygiene tasks to 2--3 months for more complex behaviour like regularly going to the gym.

% Why REBORN is needed
Most importantly, no current app makes use of this kind of context data (e.g., time lags, or day-of-week streaks) to predict user behaviour.

\subsubsection{Existing Systems}
This section analyses the four most popular habit tracking and task management apps: Notion, Habitica, Streaks, and Todoist. While other applications exist, these four represent the distinct architectural archetypes currently dominating the market.

% ---------------------------------------
\newpage
\paragraph{Description and Functionality}

\textbf{Notion} \\
Notion is a workspace that acts like a relational database wrapper. The user defines schemas (tables) and views (Kanban, Calendar) manually. The backend uses a SQL-like structure which is then served to a React frontend.
\begin{figure}[H]
    \centering
    \includegraphics[width=0.8\textwidth, trim=0 50 0 50, clip]{images/notion_secondary_research.png} 
    \caption{A Notion Habit Tracker showing the manual checkbox system.}
    \label{fig:notion}
\end{figure}

\noindent\textbf{Habitica} \\
Habitica is a gamified task manager that uses operant conditioning \cite{skinner1938}. It wraps a CRUD system in RPG mechanics, where doing tasks gives \emph{Gold/XP} and missing them causes \emph{Health Loss}.
\begin{figure}[H]
    \centering
    \includegraphics[width=0.8\textwidth]{images/habitica_secondary_research.png} 
    \caption{The Habitica dashboard showing the RPG avatar and health bars.}
    \label{fig:habitica}
\end{figure}

\noindent\textbf{Streaks} \\
Streaks is a minimalist habit tracker integrated into the Apple ecosystem. Unlike database tools, it focuses entirely on the \emph{Don't Break the Chain} psychological method \cite{clear2018atomic}. Technically, it is notable for its API integration with Apple HealthKit \cite{apple2025healthkit}, allowing it to automatically mark habits (e.g., \enquote{Walk 5,000 steps}) as complete without user input.
\begin{figure}[H]
    \centering
    \includegraphics[width=0.8\textwidth]{images/streaks_secondary_research.jpeg} 
    \caption{The Streaks interface showing its main features.}
    \label{fig:streaks}
\end{figure}

\noindent\textbf{Todoist} \\
Todoist is a task management system, designed for one-off tasks. In earlier versions, Todoist experimented with automated rescheduling through its \textit{Smart Schedule} feature, which attempted to suggest new due dates for overdue tasks. Although it was discontinued because of accuracy issues \cite{todoist2020smartschedule}, third-party integrations like Trevor AI \cite{trevorai2025} can provide similar functionality, which apply heuristic time-blocking and NLP to parse tasks and suggest times for them based on available calendar gaps.
\begin{figure}[H]
    \centering
    \includegraphics[width=0.8\textwidth, trim=0 100 0 150, clip]{images/todoist_secondary_research.jpg} 
    \caption{Todoist's ``Smart Schedule'' interface suggesting times for tasks \cite{todoist_smart_schedule_image_2016}.}
    \label{fig:todoist}
\end{figure}

\noindent\textbf{Streaks}
\begin{table}[H]
    \centering
    \begin{tabular}{|p{0.45\linewidth}|p{0.45\linewidth}|}
    \hline
    \textbf{Advantages} & \textbf{Disadvantages} \\
    \hline
    \textbf{API Automation:} Integration with Apple HealthKit removes \emph{administrative friction} by tracking physical habits automatically. & \textbf{\emph{All-or-Nothing}:} It uses binary logic. If the user misses one day, the streak resets to zero. This often causes the \emph{What-the-Hell Effect} \cite{polivy1975perceived}, where users abandon the app after a failure. \\
    \hline
    \textbf{Visual Feedback:} The circular UI provides visual status of daily progress, using \emph{loss aversion} for engagement \cite{kahneman2011thinking}. & \textbf{No Scheduling Logic:} The app tracks \textit{completion}, but not \textit{allocation}. It cannot help the user find time in a busy schedule to actually perform the habit. \\
    \hline
    Minimalist design reduces cognitive load compared to complex tools like Notion. & \textbf{No adaptive difficulty:} The target remains static regardless of user burnout. \\
    \hline
    \end{tabular}
\end{table}

\newpage
\noindent\textbf{Habitica}
\begin{table}[H]
    \centering
    \begin{tabular}{|p{0.45\linewidth}|p{0.45\linewidth}|}
    \hline
    \textbf{Advantages} & \textbf{Disadvantages} \\
    \hline
    Gamification (XP/Gold) uses dopamine loops \cite{eyal2014hooked} to increase engagement. & \textbf{Static Recurrence:} It relies on fixed rules (e.g., \enquote{Every Mon/Tue}) instead of adaptive scheduling. \\
    \hline
    Social accountability features (parties/guilds) help with external motivation. & \textbf{Reactive Punishment:} It creates a \emph{death spiral}. If a user is busy and misses tasks, the game punishes them, demotivating them further. \\

    Visualisation of progress from avatar customisation and item unlocks shows long-term habit progress. & No awareness of external schedules or calendar conflicts. \\
    \hline
    \end{tabular}
\end{table}

\noindent\textbf{Todoist}
\begin{table}[H]
    \centering
    \begin{tabular}{|p{0.45\linewidth}|p{0.45\linewidth}|}
    \hline
    \textbf{Advantages} & \textbf{Disadvantages} \\
    \hline
    NLP (Natural Language Processing) allows for quick task entry using natural language \cite{todoistQuickAdd}. & \textbf{Task vs. Habit:} The system focuses on one-off tasks and does not have a risk or failure model for long-term habit formation. \\

    \end{tabular}
\end{table}

\paragraph{Implications for Proposed System}

By investigating these systems, I have identified specific features to adopt and flaws to improve upon in the proposed solution.

\newpage
\noindent\textbf{From Notion:} \\
While a relational database is important for storing habit history, relying on the user to manually schedule every event will likely lead to burnout.
\begin{itemize}
    \item \textbf{I will include:} A strong relational database backend similar to Notion's architecture for data integrity.
    \item \textbf{I will improve:} Instead of manual entry, I will implement a Constraint Satisfaction Scheduler to automate the data entry process, solving the \emph{Blank Slate} problem.
\end{itemize}

\vspace{0.5em}
\noindent\textbf{From Habitica:} \\
Feedback mechanisms are effective, but \enquote{punishment} (losing HP) is flawed if the user is busy or overwhelmed.
\begin{itemize}
    \item \textbf{I will include:} A feedback loop system, but instead of RPG stats, I will use a \textbf{Simulation} to visualise habit stability.
    \item \textbf{I will improve:} The system will be predictive rather than reactive. By using \textbf{Logistic Regression}, my system will warn the user of risk \textit{before} they fail, instead of punishing them \textit{after}.
\end{itemize}

\vspace{0.5em}
\noindent\textbf{From Streaks:} \\
Visualising progress is very important, but the \emph{streak counter} model is too fragile for students with variable workloads.
\begin{itemize}
    \item \textbf{I will include:} A strong visual representation of habit health, similar to the \emph{rings} in Streaks, to provide immediate feedback.
    \item \textbf{I will improve:} Instead of a binary \emph{streak} that breaks (0 or 1), I will use my \textbf{simulation}. This treats habit consistency as a continuous variable (stability). If a user misses a day, the stability decays instead of resetting to zero, which is mathematically more forgiving and encourages the user to recover.
\end{itemize}

\vspace{0.5em}
\noindent\textbf{From Todoist:} \\
Algorithmic scheduling is desirable, but fixed heuristics fail when psychological factors and user failure are not modelled.
\begin{itemize}
    \item \textbf{I will include:} Automated time-blocking to ensure habits and tasks are placed within realistic daily constraints.
    \item \textbf{I will improve:} The use of a \emph{Spaced Repetition} algorithm to dynamically reduce habit frequency when users struggle, preventing overload and burnout.
    \item \textbf{I will avoid:} Set heuristic scheduling, by integrating adaptive control and predictive modelling to maintain long-term habit.
\end{itemize}

\subsubsection{Identification of Technical Gaps}

The comparison of existing tools shows three algorithmic gaps in the current market that \textbf{REBORN} aims to solve:

\begin{enumerate}
    \item \textbf{Constraint Satisfaction Solvers (CSP):} 
    Current apps rely on manual time-picking. No system treats the daily schedule as a combinatorial search problem that can be solved using backtracking or constraint propagation to guarantee conflict-free allocation \cite{russell2020ai}.
    
    \item \textbf{Predictive Logistic Regression:} 
    Applications are reactive; they show historical graphs instead of forecasting. None use the PCS framework's approach (Logistic Regression) to calculate a probability $P(fail)$ for the upcoming day to trigger preventative interventions.
    
    \item \textbf{Static Frequency vs. Adaptive Control:} 
    Habit trackers use static schedules (e.g., ``Daily''). They do not implement control loops (like the SM-2 spaced repetition algorithm \cite{wozniak1990sm2}) to adjust frequency based on user performance, for example, by scaling back load to prevent burnout.
\end{enumerate}
